  • Separación estricta de capas

  • Casos de uso claros y testeables
  • Infraestructura desacoplada

05 / 10
Microservicios e IA (RAG real)
  • OpenAI Service: gateway controlado hacia OpenAI API (claves centralizadas, CORS)
  • RAG Service: implementa Retrieval-Augmented Generation real
  • Flujo RAG: Pregunta → Retriever (KB) → Contextos → LLM → Respuesta
  • Dos modos: léxico (bag-of-words) o vectorial (embeddings + coseno)
  • Fallback automático: si falla vectorial, usa léxico sin interrumpir
  • Marvel Agent: RAG técnico que responde sobre el propio proyecto
  • La IA se integra desacoplada, no como núcleo del dominio

08 / 10
Reflexión final
Este proyecto nació con la idea de convertirse en una base de trabajo y una guía personal para aplicar, de forma real, los conocimientos adquiridos y utilizarlos como punto de partida en futuros proyectos. Comenzó como un espacio práctico donde ir trasladando lo aprendido al código, casi como un cuaderno técnico llevado a la práctica, con el objetivo de comprender en profundidad cada concepto y su impacto real en un sistema. Con el tiempo, ese ejercicio constante fue tomando forma de sistema. Un proyecto real, al que dediqué muchas horas, y que evolucionó en paralelo a mi manera de entender el desarrollo de software. Antes de este máster, mi enfoque estaba centrado principalmente en que el software funcionara. Hoy, este proyecto refleja un cambio claro: estructura, criterio, arquitectura, seguridad y responsabilidad técnica forman parte del proceso desde el inicio. Clean Marvel Album no pretende ser un producto comercial ni una solución disruptiva. Es, ante todo, una base sólida y reutilizable, pensada como referencia para desarrollar futuros proyectos con una mentalidad más madura y profesional. Este proyecto no responde a una carrera contra el calendario, sino al cierre natural de un proceso de aprendizaje intenso y consciente desarrollado a lo largo del máster. Más allá de las herramientas aprendidas, este máster ha supuesto un punto de inflexión en mi forma de pensar, diseñar y construir software. Y este proyecto es la evidencia más honesta de ese recorrido.
